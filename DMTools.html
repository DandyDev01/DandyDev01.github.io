DM Tools

About

This is a project made with WPF. It functions the same as Character Manager. Currently the program supports the creation of combat encounters which you add players, monsters and NPCs too. It some with monster creation built in and keeps track of initiative and conditions. This project is in early development.

This was the second project I made in WPF so my main focus here was the design of the UI and to make effective use of my white space.

The reason I created this program was because my DM was using discord to keep track of things. Sure there are other systems out there, but I wanted to create something.

User Interface Design Process

The first step I took was to define what the problem I was trying to solve was - Keeping track of combat encounters in D&D 5e.
With that out of the way, it was time for some requirements gathering.

  • A way to keep track of initiative.
  • Easy access to NPC and PC information relevant to combat.
    • Health.
    • Armor Class.
    • Conditions.
    • Movement Speed.
    • Attacks.
    • General Stats.
  • Way to add monsters to the encounter.
  • Way to create encounters.
  • etc.

After gathering the requirements I made a mockup for the for the user interface in Visual Studio with XAML. TO BE CONTINUED.

Software Design Process

Because I was creating the system with Windows Presentation Foundation (WPF), I already knew it would be using the Model, View, View Model (MVVM) design pattern. The biggest thing I needed to figure out was how to handle having players, monster stat blocks, and Nonplayer characters. I chose to use a combination of inheritance and composition, this let me get the best of both worlds. Composition removed large inheritance trees and a one to two layers of inheritance removed lots of duplicate code.

Putting it All Together

Now that I had a mocked UI and knew what the systems architecture would look like I was able to put together an early Alpha version of the program.

Next Steps

The next steps for the program are to design and test a final version of the user interface. This is now the center piece of the program as having an easy to use interface key.
